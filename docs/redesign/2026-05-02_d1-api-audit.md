### Step 2 — Add 6 new router modules

For each, follow the existing `api.py` patterns:
- `from fastapi import APIRouter, Depends, HTTPException`
- Reuse `_serialize`, `_clean_scalar`, `_normalize_pair` helpers from
  `api.py` — extract these to a `routers/utils.py` once a second
  router needs them.
- Reuse `require_api_key` dependency where mutation/auth is needed.
- Wrap existing engine modules:
  - `regimes.py` → `from crypto_model_core import detect_hmm_regime, ...`
  - `onchain.py` → `from data_feeds import fetch_onchain_*`
  - `alerts.py` → `from alerts import ...`
  - `ai_assistant.py` → `from llm_analysis import ...`
  - `settings.py` → `from config import ...`
- Pydantic models for POST/PUT payloads (validation + auto-doc).

### Step 3 — Wire routers in `api.py`

After the existing `app = FastAPI(...)` block, add:

```python

from routers import ai_assistant, settings as settings_router
from routers import home

app.include_router(home.router,           prefix="",          tags=["Home"])

```

## Notes for Code

- **Don't refactor `api.py` itself in D1.** It works, ships, and has
  auth/security hardening that took a P0 audit to land. New routers
  add capability without touching existing behavior.
- **Helper extraction can wait.** First copy `_serialize` /
  `_clean_scalar` into `routers/utils.py` only if a router needs them
  — don't proactively refactor.
- **AI Assistant endpoint is async-heavy** (LLM calls). Use
  `async def` + `await` on the `llm_analysis` calls; wrap the
  blocking parts in `asyncio.to_thread` if needed.
- **Settings endpoints** must persist via `config.py` patterns — do
  not bypass and write JSON directly.
- **Alerts CRUD** has SOX-like discipline (every change logged). Use
  the existing `alerts.log_alert_change()` pattern from `alerts.py`.
- **Don't add a database migration in D1.** Use existing tables.
  Schema changes (if any) go in a separate D-extension batch.

## Out of scope for D1

- Authentication beyond the existing `require_api_key` for mutations
  (NextAuth + JWT come in a post-D8 batch).
- Rate limiting (existing `data_feeds` cache already throttles
  upstream; per-endpoint rate limit is post-D8).
- WebSocket endpoints for live updates (Tanstack Query polling at
  `staleTime: 5 * 60 * 1000` is sufficient for D1-D8; WS is post-D8).
